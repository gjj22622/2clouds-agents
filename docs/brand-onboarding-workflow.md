# Brand Onboarding Workflow — 新人進入甲方品牌日常任務

本文件定義新人從「完成訓練任務」轉移到「承接真實甲方品牌日常工作」的完整流程，包含進場資格、品牌腦閱讀順序、資深成員脈絡交接、任務分派路徑、品管標準，以及營收貢獻判斷。

---

## 1. 進場資格條件

新人必須滿足以下條件，才能被分配到真實甲方品牌的日常任務：

| 條件 | 說明 |
|---|---|
| 訓練 Stage 1-3 已完成 | 讀懂双云、讀懂品牌腦、完成第一個內容任務，且均已 reviewed |
| 至少 1 次退件修稿通過 | 曾收到 needs_revision 並修正後通過，代表理解品管回饋機制 |
| 60 分認證進度 ≥ 40% | reviewedPoints / targetPoints ≥ 40%（可被視為低風險任務承接人） |
| Jacky 或 Sophia 確認 | 由資深成員判斷新人是否準備好進入真實品牌環境 |

**注意：** 進入真實品牌不代表脫離品管。所有新人輸出仍須經過 reviewer（藝嘉）審查才能交付。

---

## 2. 品牌腦閱讀順序

第一次接觸一個甲方品牌時，必須按以下順序閱讀品牌包：

### Step 1：老闆 Agent（Boss Agent）

**目的：** 理解客戶老闆的個性、決策風格與核心期望。

閱讀重點：
- 老闆的溝通語氣偏好（正式 / 非正式）
- 他在意的結果是什麼（品牌曝光、業績轉換、粉絲互動？）
- 哪些話在他面前不能說（紅線）
- 他習慣如何接收報告（LINE 語音 / 正式簡報 / 週報）

完成標準：能回答「如果老闆今天問你這個月成效如何，你會怎麼說？」

### Step 2：主管 Agent（Manager Agent）

**目的：** 理解品牌內部的工作節奏與 Sophia 或 Jacky 如何管理這個品牌。

閱讀重點：
- 目前品牌的內容節奏是什麼（幾則 / 週）
- 目前有哪些進行中的專案
- Sophia 如何和這個品牌聯絡（哪個管道、多久一次）
- 哪些工作由双云主導，哪些由客戶自行操作

完成標準：能說出「這個品牌目前的任務優先序」。

### Step 3：窗口 Agent（Contact Agent）

**目的：** 理解你作為新人，如何在日常工作中和品牌窗口協作。

閱讀重點：
- 窗口的個性與溝通習慣
- 哪些問題可以直接問窗口、哪些要先內部確認
- 窗口給的素材通常是什麼格式
- 溝通記錄應如何留存

完成標準：知道「遇到這個問題，我應該問誰、怎麼問」。

### Step 4：品牌 Agent（Brand Agent）

**目的：** 理解品牌的內容規格，這是你生產任何內容的依據。

閱讀重點：
- 品牌語氣（可用詞 / 不可用詞 / 禁忌詞）
- 每個頻道的 CTA 規範
- 視覺規格與字體禁忌（如有）
- 話題禁忌（不能提的競品、議題、事件）
- 什麼情況下內容要升級品管確認

完成標準：能填完「品牌速查卡」（見 Section 3）。

---

## 3. 品牌速查卡（Brand Context Card）

品牌速查卡是新人閱讀完品牌包後，自行整理的一頁摘要。它不是抄寫品牌包，而是「你對這個品牌的工作理解」。

每份速查卡包含 4 個區塊：

```
1. 品牌核心印象
   一句話描述：這個品牌在做什麼、對誰說、想達到什麼目標。

2. 內容語氣禁忌
   三行以內：哪些詞不能用、哪些語氣不對、哪些話題要迴避。

3. 頻道規則摘要
   各頻道的 CTA 風格差異、適合的內容長度與格式。

4. 升級判斷標準
   哪些情況我要先問藝嘉、哪些我要問 Sophia、哪些我要問 Jacky。
```

速查卡在新人進入品牌的**第一天**完成，送給藝嘉確認後才能開始承接任務。

## 4. 資深成員脈絡交接

品牌包涵蓋不了的「機構記憶」，需要由資深成員口頭或文字交接。

### 4.1 Sophia 交接的內容

Sophia 負責客戶關係，交接重點：
- 客戶目前在接案 Stage 幾（Stage 0-9）
- 最近有沒有特殊事件（客戶抱怨、合約爭議、成效下滑）
- 客戶的情緒狀態（滿意 / 觀望 / 不穩定）
- 接下來 1-2 週的重要節點（提案、月報、上線）

### 4.2 藝嘉交接的內容

藝嘉負責品管，交接重點：
- 這個品牌過去退件頻率與常見原因
- 哪些類型的內容特別容易踩到品牌禁忌
- 目前有哪些待審的草稿或修稿
- 品牌最近有沒有新的內容規格更新

### 4.3 交接完成的驗收標準

新人在收完交接後，必須能回答：
1. 這個品牌現在最需要解決的問題是什麼？
2. 接下來 1 週，我預計要完成哪些任務？
3. 什麼情況下我需要先問 Sophia，什麼情況下先問 Jacky？

交接回答由 Sophia 或 Jacky 確認，確認後才算完成進場。

## 5. 任務分派路徑：低風險 → 高風險

新人進入真實品牌後，任務依風險層級分配：

### 低風險任務（新人可獨立完成，藝嘉必審）

- 草稿檢查：核對一則現成草稿是否符合品牌腦
- 素材整理：整理客戶提供的圖片、文案素材，補充缺口清單
- 前情提要：整理一個內容任務的 brief，送給藝嘉確認再執行
- 合規初查：初步掃描即將發布的內容，標出合規疑點

### 中風險任務（新人初稿 + 藝嘉全審）

- 社群短文：FB / IG 一般貼文，新人初稿後由藝嘉逐字審查
- 週報數據整理：整理週報數字與基本說明，資深成員確認解讀方向

### 高風險任務（新人不可獨立完成）

以下任務**不可由新人獨立執行**，必須有資深成員主責：
- 客戶月報與策略回顧
- 新 campaign 方向提案
- 危機回應文
- 任何需要對外直接承諾的內容
- 合約、報價相關的回應

**判斷原則：** 若任務的輸出會直接影響客戶對双云的信任，或涉及策略承諾，則屬於高風險。

## 6. 品管標準的差異：訓練任務 vs. 真實品牌任務

| 面向 | 訓練任務 | 真實品牌任務 |
|---|---|---|
| 品管嚴格度 | 有容錯空間，退件是訓練 | 退件仍是回饋，但不能影響交付時程 |
| Reviewer | 藝嘉審查 | 藝嘉審查（必須） |
| Reviewer note | 以訓練為目的 | 以保護品牌與客戶關係為目的 |
| Trace log | 新人自行操作即可 | 每個步驟都要有清楚的 trace |
| 升級標準 | 遇到不確定就升級 | 同左，但要更快判斷、更少讓資深成員等 |
| 時效要求 | 無硬性時限 | 依客戶服務 SLA，通常 24-48 小時內 |

## 7. 營收貢獻判斷

新人承接真實品牌任務時，要了解自己的工作如何影響实際客戶服務交付。

### 7.1 任務 → 交付 → 收費的連結

每個真實品牌任務最終對應到一個收費交付物（如：月報一份、IG 貼文 10 則）。新人的工作是這條鏈條的一環：

```
新人任務（草稿 / 前情提要 / 素材整理）
    ↓
藝嘉品管（結構 + 品牌 + 合規）
    ↓
資深成員確認（Sophia / Jacky）
    ↓
客戶交付（可計費成果）
```

### 7.2 新人任務的品質如何影響營收

- **品質穩定** → 減少資深成員修稿時間 → 提高服務效率 → 維持客戶續約意願
- **品質不穩定** → 增加退修次數 → 資深成員介入成本提高 → 影響服務利潤率
- **流程規範** → trace log 完整、升級判斷準確 → 降低溝通成本 → 提升服務品質形象

### 7.3 什麼叫「新人任務貢獻了營收」

以下情況可認定新人任務有直接或間接的營收貢獻：
- 新人初稿在「不超過 1 次退修」內通過品管，資深成員僅確認不需重寫
- 新人的前情提要正確框定了任務，讓內容執行減少來回確認次數
- 新人標出的升級問題讓 Sophia 能在下次客戶會議中準時回應

新人進入真實品牌 4 週後，Sophia 與藝嘉會共同評估：是否可以承接更多中風險任務。

### 7.4 RevenueSignal 的建立時機與責任人

`RevenueSignal` 代表與品牌客戶關係相關的業務訊號。**新人不建立、不修改**此記錄；由 Sophia 或 Jacky 在下列時機手動建立：

| 時機 | 訊號類型 | confidence 建議 |
|---|---|---|
| 新客戶第一次付款或合約確認 | `conversion` | high |
| 客戶月報後確認繼續合作 | `retention` | high |
| 客戶詢問增加服務項目 | `upsell` | medium |
| 客戶提出不滿或要求改善 | `retention` | low |
| 新客戶諮詢但尚未確認 | `lead` | low 或 medium |

建立後，`BrandTask.revenueSignalIds` 由資深成員填入，標記「哪個任務與這個營收訊號直接相關」。新人看到 `revenueSignalIds` 不為空，代表這個任務有業務意義，需要更謹慎執行。

### 7.5 BrandTask 與 RevenueSignal 的連結邏輯

```text
BrandTask（由資深成員建立）
  └── revenueSignalIds: ["rev-001", "rev-002"]
        │
        ├── rev-001（type: conversion, confidence: high）
        │     └── 代表：這個任務的交付成果直接對應本月收費
        │
        └── rev-002（type: retention, confidence: medium）
              └── 代表：這個任務的品質會影響客戶續約判斷
```

新人判斷自己任務的業務重要性，可參考：
- `revenueSignalIds` 數量越多 → 任務對業務影響越直接
- 包含 `confidence: high` 的訊號 → 屬於高優先任務，不可超時
- 包含 `type: retention` 且 `confidence: low` → 客戶關係敏感期，語氣與品質需特別謹慎

**新人行動原則：** 看到 `revenueSignalIds` 不為空時，視任務風險等級提升一級處理。即使是原本歸類為「低風險」的草稿任務，若連結到 high-confidence 的 `conversion` 訊號，也應主動詢問 Sophia 是否需要額外確認。

## 8. 與訓練 Stage 6 認證的連結

真實品牌低風險任務可計入 Stage 6「60 分認證」的實戰項目，條件是：
- 任務已由 reviewer 標記為 reviewed
- Trace log 完整記錄操作過程
- 升級判斷符合認證標準（在該升級時升級、不必要時不打擾資深成員）

Stage 6 認證完成後，Jacky 決定是否正式將新人列為某品牌的「日常任務承接人」。

## 9. 第一天工作流程（Day 1 Checklist）

以下是新人被指派到真實品牌的第一天標準流程。每個步驟都要在平台上留下 trace log。

**前提：** Sophia 已完成進場 Handoff 說明（SeniorMemberActivity type: handoff 已建立）。

### 上午：品牌腦閱讀（2-3 小時）

```
□ Step 1：開啟品牌工作台，確認 Brand App 已完成資料填寫
   → 確認 BrandBrain 的 voice / taboos / channelRules 都有內容
   → 確認有至少 1 筆 SeniorMemberActivity（handoff）可閱讀

□ Step 2：依序閱讀品牌腦四個 Agent（見 Section 2）
   老闆 Agent → 主管 Agent → 窗口 Agent → 品牌 Agent
   每個 Agent 閱讀完記錄 1 個「你認為最重要的工作行為原則」

□ Step 3：閱讀 SeniorMemberActivity 的 Handoff 記錄
   理解品牌目前狀態、近期任務重點、注意事項
```

### 午前：品牌速查卡（1-1.5 小時）

```
□ Step 4：開啟「閱讀品牌包並整理品牌速查卡」任務（task-brand-context-card）
   填寫 4 個區塊：
   ① 品牌核心印象（1 句話）
   ② 語氣禁忌（3 點以內）
   ③ 頻道規則差異（列主力平台）
   ④ 升級判斷標準（分 Sophia / Jacky / 藝嘉）

□ Step 5：完成後送出（submitted），等藝嘉確認
   → 在送出前先自我品管：對照品牌腦確認沒有漏掉任何禁忌
   → 在 trace log 留下「完成首次品牌速查卡」的備注
```

### 午後：資深成員確認（等待中）

```
□ Step 6：等待藝嘉的速查卡審查結果
   → 如果收到 needs_revision，看清楚 reviewer note，修正後重新送出
   → 速查卡通過 (reviewed) 前，不開始任何品牌任務

□ Step 7：如速查卡當天未能確認，利用等待時間：
   → 閱讀平台的 kg-brand-onboarding 知識節點
   → 看一遍品牌的 RevenueSignal，了解品牌目前的業務信號
   → 整理 3 個「明天想問 Sophia 的問題」（不要現在問，集中提問）
```

### 傍晚：確認第一個任務（速查卡通過後）

```
□ Step 8：Sophia 或藝嘉分配第一個低風險任務
   → 確認任務的 expectedOutcome
   → 確認任務屬於低風險（草稿檢查 / 素材整理 / 前情提要）
   → 如果任務屬於中高風險，告知 Sophia 說明你目前的能力邊界

□ Step 9：第一天結束前，記錄今日工作摘要到 trace log
   → 哪些品牌腦節點已讀完
   → 速查卡的審查結果
   → 第一個任務的啟動狀態
   → 有沒有不確定的問題需要明天問 Sophia

□ Step 10：將集中的問題用文字整理，傳給 Sophia（LINE 或內部管道）
   → 格式：「問題 + 你目前的理解 + 你想確認什麼」
   → 不要只問「我不知道怎麼做」，要說明你的困惑點
```

### 第一天完成標準

| 確認項目 | 完成條件 |
|---|---|
| 品牌腦閱讀 | 四個 Agent 都閱讀過，每個有 1 點重點記錄 |
| 速查卡送出 | 已標記為 submitted 或 reviewed |
| Handoff 記錄閱讀 | SeniorMemberActivity 已閱讀 |
| 問題整理 | 至少 3 個問題已整理成文字傳給 Sophia |
| Trace log | 今日有至少 1 筆 trace 記錄 |

**如果速查卡當天未能通過審查（藝嘉較忙），第一天的完成標準仍以「送出速查卡」為準。不要因為等待就停止準備。**
