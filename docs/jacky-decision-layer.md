# Jacky 知識圖譜與決策層

## 1. 核心定位

Jacky Decision Layer 是平台的判斷中樞。

它不是單純 FAQ，也不是一般聊天機器人，而是在成員卡住時，根據 Jacky Wiki 的知識圖譜、方法論與思考模型，協助成員做出最佳決策與回應。

## 2. 使用情境

成員可以在任何工作節點呼叫：

```text
問 Jacky
```

或：

```text
用 Jacky 模型判斷
```

典型情境：

- 客戶問：「我們為什麼要做 AI 行銷部？」
- 新人不知道這篇文案能不能發
- Sophia 接案時不確定這個客戶該不該接
- 藝嘉審稿時不確定這算品牌偏差還是合規問題
- 成員不知道該用 SOSTAC、SPEAKS 還是 AGENTS
- 客戶質疑報價、成效、服務範圍
- 內容策略不知道該往品牌、轉換、教育還是信任走

## 3. 回應格式

Jacky Decision Layer 每次跳出來，不只給答案，而是提供判斷支架。

### 3.1 問題重述

- 你現在其實遇到的是什麼問題？
- 是策略問題、執行問題、客戶溝通問題、品牌一致性問題，還是流程問題？

### 3.2 對應知識節點

- 來自 wiki 哪些頁
- 屬於哪個方法論
- 相關案例或 SOP
- 關聯品牌腦或成員腦

### 3.3 思考模型

可使用模型：

- SOSTAC：現況、目標、策略、戰術、行動、控制
- AGENTS：拆、建、修、串、管、交
- AI 行銷部：老闆 1-4，AI 5-8
- 4 Agent 委員會：老闆、主管、窗口、品牌
- Brain / Hand / Infra：誰判斷、誰執行、誰記錄
- 殘酷測試：標準、邊緣、錯誤情境

### 3.4 建議決策

- 建議怎麼判斷
- 建議怎麼回客戶
- 哪些話不能說
- 哪些資訊不足，需要補問
- 什麼情況要升級給 Jacky / Sophia

### 3.5 可直接使用的回應草稿

- 對客戶版本
- 對內部團隊版本
- 簡短 LINE 版本
- 正式提案版本

## 4. 知識圖譜節點

知識節點不是普通文章，而是可被工作流呼叫的決策物件。

每個節點包含：

- 概念
- 適用情境
- 不適用情境
- 關聯節點
- 可回答的問題
- 可產生的輸出
- 代表性話術
- 對應平台功能
- 來源 wiki 頁面

範例：

```text
AI 行銷部
→ 連到：老闆 1-4 / AI 5-8
→ 連到：4 Agent 委員會
→ 連到：內容工廠
→ 連到：客戶導入
→ 連到：品牌工作台
→ 可回答：我們跟一般代操差在哪？
→ 可判斷：這個客戶適不適合導入 AI 行銷部？
```

## 5. 決策輔助側欄

每個任務頁右側固定一個 Jacky Decision Panel。

內容：

- 目前任務判斷點
- 推薦模型
- 相關 wiki 節點
- 可問 Jacky 的問題
- 可能風險
- 建議下一步
- 升級條件

## 6. 方法論散布原則

方法論不能只放在 `/training`。

它應散布在：

- 客戶導入節點
- 品牌工作台
- 內容工廠
- 品管中心
- 新人訓練場
- Skill OS
- 自動化與 API

平台要在成員需要判斷時，把正確模型推到他面前。
